织梦移动端生成静态页的开发

织梦移动端生成静态页的开发

1.安装织梦官方的wap插件

此处不多叙述，wap插件很简单，一个php配置文件，一个php执行文件、自己写一个也行，注意参数过滤就可以，关于参数过滤可以参考织梦根目录plus里面的文件。

安装完之后，可以给wap.php改个名字（为了安全、也为了好看，可以改成mobile.php）

织梦模板早templets/wap/下面，模板写的很差，可以自己重新写html模板，模板采用php html 混编，
如果模板很复杂建议使用php定界符

2.修改wap.php

这个php写的不正规，参数过滤也有点不过关，引用织梦核心配置文件不建议使用相对路径(dirname(__FILE__))这种方式，可以用织梦全局常量DEDEINC(织梦include文件夹的路径)等。

  • 根据当前文章id获取该文章的静态页路径（PC端）

参考织梦dede/archives_do.php 这个文件的 $dopost==”viewArchives” 这个执行片段，其中执行的是$trow[‘issystem’] != -1 这个条件语句，最后可以得到$arcurl

整个wap.php文件都是各种判断语句，查询语句，输出到模板，可以根据自己的需求自己编写。

目前整个路径都是action=list&id=123很难看，可以rewrite一下，

以下是nginx的rewrite

location / {
    rewrite ^/(.*)/(.*)$ /wap.php?action=$1&id=$2 last;
}

3.mobile端静态化

这时我们就可以不用上面wap.php这个文件了，由于生成静态文件需要在管理端进行，我们需要到管理目录（默认dede）新建一个wap_php_html.php的生成文件，可以把刚刚wap.php的代码copy过来

头部加上：

require_once(dirname(__FILE__).'/config.php');

//用来加载管理目录的类，以及是否登录的判断

接下来就在每个action末尾添加

//更新主页

$dsql->Close();
$fp = fopen('../**.html', 'w'); 
fwrite($fp,$pageBody);
fclose($fp);
exit('更新主页成功');

$fp = fopen('../articleHtml/'.$id.'.html', 'w'); //articleHtml这是个目录，要事先建好，给权限
fwrite($fp,$pageBody);
fclose($fp);
exit('更新主页成功');

//一键更新全站
//定义开始文章id为1 $aid=1

if ($action == 'makeAll')
{
    $maxid = $dsql->GetOne("SELECT id FROM `#@__arctiny`  order by id desc limit 0,1 $idsql" );//获取文章最大id

$max_num = $maxid['id'];

for($id=$aid;$id<=($aid+20);$id++){
//一次请求更新20篇文章
    这里做数据处理，和action='article'里面的语句差不多一致，根据$id 查询出数据库的数据
    $dsql->Close();
    $fp = fopen('../articleHtml/'.$id.'.html', 'w'); //articleHtml这是个目录，要事先建好，给权限

用hexo搭建的博客

首先这个博客已经筹备很久了，这几天才弄起来，实在是有些惭愧啊。以此纪念我的hexo搭建的静态博客。目前page托管在github上；

在搭建博客的过程中，首先感谢下面两个博主:

http://zipperary.com/categories/hexo/
http://blog.csdn.net/poem_of_sunshine/article/details/29369785/

1. hexo搭建

本地环境：mac、git、node

安装hexo

npm install -g hexo
hexo init
npm install

安装完之后，本地启动服务

hexo generate
hexo server

这样在本地就可以通过http://0.0.0.0:4000查看了，hexo服务启动之后，不能关闭，否则本地无法参加查看

2. 配置git托管page

  1. 配置ssh 避免每次提交都输入git的账号密码

查看是否有以前的key,有责备份

cd ~/.ssh

mkdir key_backup

mv id_rsa* key_backup

设置git账号信息

git config --global user.name "你的git账号"

git config --global user.email "你的邮箱"

生成SSH key

ssh-keygen -t rsa -C "你的邮箱”

Enter file in which to save the key (/Users/helonghua/.ssh/id_rsa):直接回车就

复制~/.ssh/id_rsa 这里面的密钥，为了避免格式错误，建议用sublime打开

登录到GitHub页面，Account Settings->SSH Public Keys->Add another key将生成的key(id_rsa.pub文件）内容copy到输入框中，save。

测试链接

ssh git@github.com

设置托管git的路径 编辑根目录_config.yml 【很重要以上5步都可以先不做】

deploy: 
    type: git 
    repo: git@github.com:xxxx/xxxx.github.io.git 
    branch: master

提交至git

hexo generate
hexo deploy

这样就ok了，yidiandianlan.github.io可以查看

3.创建新的文档

hexo new "My New Post"

创建名为My-New-Post的项目 在source/_post下面生成My-New-Post.MD

4.配置自己的域名

在source目录下创建文件CNAME 写入 helonghua.com 即可

上传至git，再将helonghua.com 解析@.heonghua.com

  • 192.30.252.153

  • 192.30.252.154

这两个ip

5.常见问题

更换主题，下载之后放置theme下即可、修改_config.yml

theme: yilia

快捷命令

hexo g == hexo generate
hexo d == hexo deploy
hexo s == hexo server
hexo n == hexo new

快速提交上传git

sudo hexo d -g

碰见疑难杂症?

直接删除根目录的db.json
