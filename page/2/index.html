mac下重新编译php新版本

解决掉mac自带的php

我的处理方式有点暴力~ 自己删除文件，把下面的干掉基本自带的php就没有了，还不放心？find一下php

/private/etc/           sudo rm -rf php-fpm.conf.default php.ini php.ini.default
/usr/bin/               sudo rm -rf php php-config phpdoc phpize
/usr/include            sudo rm -rf php
/usr/lib                sudo rm -rf php
/usr/sbin               sudo rm -rf php-fpm
/usr/share              sudo rm -rf php
/usr/share/man/man1     sudo rm -rf php-config.1 php.1 phpize.1
/usr/share/man/man8     sudo rm -rf php-fpm.8

为什么要重新装php呢？mac下自带的php让人忍受不了，升级了一下系统，发现gd库生成图片不行了~

基本环境的安装

libpng安装

  • 下载地址：http://jaist.dl.sourceforge.net/project/libpng/libpng16/1.6.16/libpng-1.6.16.tar.xz

  • 安装

    tar zxf libpng-1.6.16.tar.xz
cd libpng-1.6.16.tar.xz
./configure --prefix=/usr/local/libpng
make
make install

jpeg安装

  • 下载地址 http://down1.chinaunix.net/distfiles/jpegsrc.v7.tar.gz 【chinaunix】下载站得资源挺靠谱。sourceforge下载还是比较慢得。
  • 安装同上，记得修改安装路径prefix

freetype安装

  • 下载地址：http://jaist.dl.sourceforge.net/project/freetype/freetype2/2.5.5/freetype-2.5.5.tar.bz2

zlib安装

  • mac自带了~~

libmcrypt安装

  • 安装php的时候提示我加密库没有，这个最开始也是下载安装包编译~，不过一直不成功，直接用brew装了

php编译

开始进入正题了、先把安装包下载回来，直接上php.net down安装包、5.6： http://cn2.php.net/distributions/php-5.6.6.tar.gz 解压…..

编译参数

参数仿照aliyun、记得把fpm参数加上（第一次没加上，重新编译一次）
其中apx2 路径需要先查看，libpng、freetype、jpeg参数都是上面编译的路径

./configure --prefix=/usr/local/php \
--with-config-file-path=/usr/local/php/etc \
--with-apxs2=/usr/sbin/apxs \
--with-mysql=mysqlnd \
--with-mysqli=mysqlnd \
--with-pdo-mysql=mysqlnd \
--enable-static \
--enable-maintainer-zts \
--enable-zend-multibyte \
--enable-inline-optimization \
--enable-sockets \
--enable-wddx \
--enable-zip \
--enable-fpm \
--with-fpm-user=www \
--with-fpm-group=www \
--enable-calendar \
--enable-bcmath \
--enable-soap \
--with-zlib \
--with-iconv \
--with-gd \
--with-xmlrpc \
--enable-mbstring \
--without-sqlite \
--with-curl \
--enable-ftp \
--with-mcrypt  \
--with-freetype-dir=/usr/local/freetype \
--with-jpeg-dir=/usr/local/jpeg \
--with-png-dir=/usr/local/libpng


make && make install

这两个参数可以在/usr/local/php/etc/php-fpm.conf里面修改

--with-fpm-user=www \
--with-fpm-group=www \

[其他常见问题http://blog.csdn.net/dc_726/article/details/9519619]

编译完后，设置php全局变量[非必须]；

我是做软链形式sudo ln -s /usr/local/php/bin/php /usr/bin/php，也可以加入全局变量$PATH

php查看模块安装

php -m | grep pdo

php-fpm 设置：

复制php-fpm配置文件 

cp /usr/local/php/etc/php-fpm.conf.default /usr/local/php/etc/php-fpm.conf

vim php-fpm.conf   

//   #pid = run/php-fpm.pid 将#号去掉

php-fpm启动

以下在mac上没做测试

【fpm开机自启动】

网上有各种版本的php-fpm开机自动启动脚本, 其实你编译后源目录已经生成自动脚本。不用做任何修改即用。

cp {php-5.6.x-source-dir}/sapi/fpm/init.d.php-fpm /etc/init.d/php-fpm
chmod +x /etc/init.d/php-fpm
chkconfig --add php-fpm
chkconfig php-fpm on
service php-fpm start

启动fpm /usr/local/php/sbin/php-fpm

kill fpm 进程

lsof -i:9000

sudo kill **

其他配置

如果你的网站全站资源都是绝对路径，那已经不需要修改什么了。如果你的图片路径是/uploads/img
这样的相对路径、同时由启用的 二级独立域名mobile.***.com

那你就需要在apache 或者nginx 中rewrite了

apache vhost配置

增加

RewriteEngine on
RewriteCond $1 ^(uploads)
RewriteRule ^/(.*)$ http://www.dede.com/$1 [L]

注意：www.dede.com 是你主站的主域名。

.htaccess

.htaccess 注意是在 /m/目录哦，不要放错地方了~

<IfModule rewrite_module>
    RewriteEngine on
    RewriteBase /
    RewriteCond $1 ^(uploads)
    RewriteRule ^(.*)$ http://www.dede.com/$1 [L]
</IfModule>

用hexo搭建的博客

首先这个博客已经筹备很久了，这几天才弄起来，实在是有些惭愧啊。以此纪念我的hexo搭建的静态博客。目前page托管在github上；

在搭建博客的过程中，首先感谢下面两个博主:

http://zipperary.com/categories/hexo/
http://blog.csdn.net/poem_of_sunshine/article/details/29369785/

1. hexo搭建

本地环境：mac、git、node

安装hexo

npm install -g hexo
hexo init
npm install

安装完之后，本地启动服务

hexo generate
hexo server

这样在本地就可以通过http://0.0.0.0:4000查看了，hexo服务启动之后，不能关闭，否则本地无法参加查看

2. 配置git托管page

  1. 配置ssh 避免每次提交都输入git的账号密码

查看是否有以前的key,有责备份

cd ~/.ssh

mkdir key_backup

mv id_rsa* key_backup

设置git账号信息

git config --global user.name "你的git账号"

git config --global user.email "你的邮箱"

生成SSH key

ssh-keygen -t rsa -C "你的邮箱”

Enter file in which to save the key (/Users/helonghua/.ssh/id_rsa):直接回车就

复制~/.ssh/id_rsa 这里面的密钥，为了避免格式错误，建议用sublime打开

登录到GitHub页面，Account Settings->SSH Public Keys->Add another key将生成的key(id_rsa.pub文件）内容copy到输入框中，save。

测试链接

ssh git@github.com

设置托管git的路径 编辑根目录_config.yml 【很重要以上5步都可以先不做】

repo 的地址就是 项目ssh的地址，注意不是https哦。

deploy: 
    type: git 
    repo: git@github.com:xxxx/xxxx.github.io.git 
    branch: master

提交至git

hexo generate
hexo deploy

这样就ok了，yidiandianlan.github.io可以查看

3.创建新的文档

hexo new "My New Post"

创建名为My-New-Post的项目 在source/_post下面生成My-New-Post.MD

4.配置自己的域名

在source目录下创建文件CNAME 写入 helonghua.com 即可

上传至git，再将helonghua.com 解析@.heonghua.com

  • 192.30.252.153

  • 192.30.252.154

这两个ip

5.常见问题

更换主题，下载之后放置theme下即可、修改_config.yml

theme: yilia

快捷命令

hexo g == hexo generate
hexo d == hexo deploy
hexo s == hexo server
hexo n == hexo new

快速提交上传git

sudo hexo d -g

碰见疑难杂症?

直接删除根目录的db.json

重装hexo

今天碰到一恶心的事，hexo提交直接卡了，也不报错，直接把hexo删了，找到node_modules/hexo

直接删了。然后把hexo的主题，配置，文档备份，重装hexo

重装过程中得问题：

1.unlink ‘/Users/helonghua/blog/.deploy_git/CNAME’
解决：权限不够，sudo

2.提示：Deployer not found: git
解决：npm install hexo-deployer-git —save

3.Permission denied (publickey).

git 公钥有问题，直接 hexo new "My New Post"
